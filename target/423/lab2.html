Lab Part 2: Lex Testing

  • Write test cases for each token category, especially the non-trivial ones.
  • Hello World is a test case (.rs file).
  • It provides coverage for ten token categories.

  • Unterminated strings
  • Random binary garbage characters
  • ...etc.

Answers to Lab Questions

Does "categories" mean the six kinds of Tokens Rust Docs talk about?
Nope, it means all the subcategories that are syntactically distinguishable
What do you mean, put the tokens in yylex()?
I mean, malloc() an instance of each token inside yylex() and initialize its fields before you return the integer category. The type definition for tokens should be in a .h file.
How do I de-escape the string literals for the svals as shown in the HW#2 example output?
You probably write a helper function and build an sval from yytext by doing a "modified copy" where most yytext characters are copied into the sval, but escaped characters in yytext map 2+ characters down to a single character in the sval.
It seems like you want a lot of stuff to be done inside yylex()
Yes, probably in 1+ helper functions called from the yylex() semantic actions code fragments.
